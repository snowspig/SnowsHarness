# Code Review Standards

## 分类标准

| 严重性      | 说明                                   |
| ----------- | -------------------------------------- |
| **Blocker** | 生产故障（安全漏洞、数据丢失、崩溃）   |
| **High**    | 近期可能出问题（竞态、缺少错误处理）   |
| **Medium**  | 应修复但不紧急（代码味、缺测试、命名） |
| **Nitpick** | 风格偏好、小可读性改进                 |

## 每个发现必须包含

- 文件路径 + 行号
- 什么问题
- 为何重要
- 建议修复
- ✅ 有证据，无发现不报告
