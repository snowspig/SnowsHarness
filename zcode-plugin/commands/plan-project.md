分析当前项目并生成/推荐 AGENTS.md 配置与开发设置。

$ARGUMENTS

## 步骤

1. **检测项目类型**（扫描代码库标识文件）：
   - Python: `pyproject.toml`, `setup.py`, `requirements.txt`, `*.py`
   - Frontend: `package.json`, `tsconfig.json`, `*.tsx`, `*.vue`
   - 全栈: 两者兼有
   - 其他: Go (`go.mod`), Rust (`Cargo.toml`), Java (`pom.xml`)

2. **分析技术栈**：框架、数据库、测试框架、构建工具、CI/CD

3. **生成项目 AGENTS.md**，包含：
   - 项目概述与技术栈
   - 常用命令（test/lint/build/run）
   - 项目结构
   - 关键约定
   - 环境配置

4. **必须包含协作原则** —— 生成的 AGENTS.md 末尾追加第一性原理协作段：

   ```markdown
   ## 协作原则

   从原始需求和问题本质出发，不从惯例和模板出发：
   - 不假设用户清楚自己想要什么 — 动机或目标不清晰时，停下来讨论
   - 目标清晰但路径不是最短的 — 直接告诉用户并建议更好的办法
   - 遇到问题追根本原因，不打补丁
   - 每个决策都要能回答"为什么"
   - 输出说重点，砍掉一切不改变决策的信息
   ```

   可按项目类型补充更具体规则，但必须保留以上 5 条。

5. **检查 AGENTS.md 是否已存在**：
   - 存在 → 基于当前状态建议更新；缺协作原则则补上
   - 不存在 → 用完整模板创建

6. **推荐 ZCode 优化**：
   - 哪些 skill 有用
   - 代码索引是否适用（code_search 等工具）
   - 自定义命令建议

输出推荐的 AGENTS.md 内容及所需的设置命令。
